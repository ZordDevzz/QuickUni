1. TAILWIND CSS (v4)
--------------------------------------------------------------------------------
Framework CSS tiện ích (utility-first) dùng để thiết kế giao diện nhanh chóng.

* ƯU ĐIỂM (PROS):
  - Biên dịch siêu nhanh (Rust Engine): Phiên bản v4 sử dụng bộ biên dịch mới viết 
    bằng Rust, tăng tốc độ build đáng kể so với các bản cũ.
  - Cấu hình trực tiếp trong CSS: Loại bỏ file `tailwind.config.js` truyền thống. 
    Mọi tùy biến theme, font, color hiện tại được thực hiện trực tiếp bằng chỉ thị 
    `@theme` trong file CSS nguồn (như [globals.css](file:///d:/source/QuickUni/webapp/quick-uni/src/app/globals.css)), giúp đồng bộ hóa tốt với các tiêu chuẩn CSS hiện đại.
  - Hỗ trợ CSS hiện đại tốt hơn: Tích hợp sẵn Container Queries, cascade layers, 
    và các thuộc tính CSS mới mà không cần plugin bổ sung.
  - Không có chi phí Runtime: Tailwind tạo ra file CSS tĩnh sau khi build, giúp 
    tải trang nhanh và không làm chậm trình duyệt của người dùng.

* NHƯỢC ĐIỂM (CONS):
  - Thay đổi lớn so với v3 (Breaking Changes): Cách viết cấu hình mới và việc loại 
    bỏ một số công cụ tiền xử lý cũ có thể gây khó khăn cho các lập trình viên đã 
    quen với Tailwind v3. Một số plugin cộng đồng chưa hoàn toàn tương thích.
  - HTML cồng kềnh (Class Soup): Các thẻ HTML/JSX chứa danh sách class rất dài, 
    gây khó khăn trong việc đọc và bảo trì cấu trúc code nếu không chia nhỏ component.

2. SHADCN/UI & RADIX UI
--------------------------------------------------------------------------------
Bộ thư viện thành phần giao diện không style (headless UI) kết hợp với các component mẫu.

* ƯU ĐIỂM (PROS):
  - Quyền kiểm soát mã nguồn 100%: Shadcn/UI không phải là một thư viện npm cài đặt 
    thông thường. Mã nguồn component được tải trực tiếp vào thư mục dự án 
    (ví dụ: `src/components/ui`). Lập trình viên có thể tùy biến trực tiếp bất kỳ 
    dòng code logic hay style nào mà không bị giới hạn.
  - Khả năng tiếp cận cao (Accessibility - WAI-ARIA): Dựa trên nền tảng Radix UI 
    primitives, đảm bảo tất cả component như Dropdown, Modal, Dialog đều hỗ trợ 
    phím mũi tên, phím ESC, và trình đọc màn hình tốt cho người khuyết tật.
  - Thẩm mỹ hiện đại và cao cấp: Thiết kế mặc định rất sạch sẽ, hỗ trợ biến số CSS 
    (CSS Variables) giúp việc đổi theme đồng bộ cực kỳ dễ dàng.

* NHƯỢC ĐIỂM (CONS):
  - Khó khăn khi cập nhật: Do mã nguồn nằm trực tiếp trong dự án, khi Shadcn/UI 
    phát hành bản vá lỗi hoặc tính năng mới cho component, lập trình viên phải tự 
    so sánh code (diff) hoặc tải đè tệp tin và cấu hình lại những phần đã tùy biến.
  - Quản lý code phình to: Việc sở hữu hàng chục file component nhỏ trong dự án 
    làm tăng số lượng tệp tin cần quản lý trong codebase.

3. GSAP (GREENSTOCK ANIMATION PLATFORM) & MOTION (FRAMER MOTION v12)
--------------------------------------------------------------------------------
Bộ đôi công cụ xử lý hiệu ứng chuyển động và tương tác giao diện.

* ƯU ĐIỂM (PROS):
  - GSAP - Sức mạnh hiệu năng vượt trội: Là tiêu chuẩn vàng cho các chuyển động phức 
    tạp, hiệu ứng cuộn trang (ScrollTrigger), tương tác SVG, và xử lý timeline. 
    GSAP tối ưu hóa phần cứng cực tốt, đảm bảo chuyển động không bị giật lag (frame drop).
  - Motion (Framer Motion v12) - Thân thiện với React: Hỗ trợ cú pháp khai báo 
    (declarative) rất trực quan trong React thông qua thẻ `<motion.div>`. Xử lý 
    tuyệt vời các hiệu ứng chuyển trang (AnimatePresence), chuyển động layout 
    (Layout animations) và các tương tác hover/tap nhẹ nhàng.

* NHƯỢC ĐIỂM (CONS):
  - Dư thừa thư viện (Redundancy): Sử dụng cả GSAP và Motion trong cùng một dự án 
    làm tăng kích thước gói tải về (bundle size) của Client. Cần có quy định rõ 
    ràng khi nào dùng GSAP (cho trang giới thiệu lớn, cuộn trang phức tạp) và khi 
    nào dùng Motion (cho các component giao diện thông thường).
  - Phức tạp khi tích hợp GSAP vào React: Việc quản lý vòng đời (cleanup khi component 
    unmount để tránh rò rỉ bộ nhớ) trong React với GSAP đòi hỏi lập trình viên phải 
    sử dụng hook `useGSAP` hoặc quản lý refs cẩn thận.

4. LUCIDE REACT
--------------------------------------------------------------------------------
Thư viện cung cấp bộ biểu tượng dạng vector (SVG).

* ƯU ĐIỂM (PROS):
  - Hỗ trợ Tree Shaking: Chỉ những biểu tượng nào thực sự được import mới được 
    đóng gói vào bản build cuối cùng, giúp tối ưu hóa dung lượng ứng dụng.
  - Đồng bộ và dễ tùy biến: Kích thước nét vẽ (stroke-width), màu sắc và kích 
    thước icon dễ dàng thay đổi thông qua Tailwind CSS classes.

* NHƯỢC ĐIỂM (CONS):
  - Giới hạn về phong cách: Hầu hết biểu tượng đều theo phong cách dạng viền (outline), 
    gây khó khăn nếu thiết kế yêu cầu các biểu tượng dạng đổ màu (filled) phức tạp 
    hoặc mang tính chất minh họa màu sắc sặc sỡ.

5. NEXT-THEMES & SONNER
--------------------------------------------------------------------------------
Các công cụ bổ trợ tiện ích giao diện (Theme & Toast).

* ƯU ĐIỂM (PROS):
  - next-themes: Tránh hiện tượng nháy sáng (flicker) khi tải trang đối với các ứng 
    dụng render trên máy chủ (SSR). Tự động nhận diện cấu hình theme hệ thống của 
    người dùng.
  - Sonner: Trình hiển thị toast siêu nhẹ, hỗ trợ cử chỉ vuốt để tắt, thiết kế đẹp 
    mắt và hiển thị được nhiều thông báo xếp chồng mà không làm rối mắt.

* NHƯỢC ĐIỂM (CONS):
  - Ảnh hưởng Client Bundle: Dù dung lượng rất nhỏ, chúng vẫn bắt buộc phải chạy 
    dưới dạng Client Component, tăng thêm một lượng nhỏ Javascript cần xử lý.

6. CLSX & TAILWIND-MERGE
--------------------------------------------------------------------------------
Tiện ích quản lý class động.

* ƯU ĐIỂM (PROS):
  - Giải quyết xung đột class Tailwind: `tailwind-merge` giải quyết triệt để lỗi 
    ghi đè class (ví dụ: `px-2` và `px-4` cùng xuất hiện thì nó sẽ tự động giữ lại 
    class sau cùng), đảm bảo style áp dụng đúng như mong đợi.
  - Viết code sạch: Kết hợp với `clsx` giúp lập trình viên viết điều kiện thêm class 
    một cách ngắn gọn và dễ đọc.

* NHƯỢC ĐIỂM (CONS):
  - Chi phí xử lý nhỏ (Runtime Overhead): Việc phân tích cú pháp chuỗi class ở 
    Client có tốn một chút thời gian chạy CPU, tuy nhiên con số này là cực kỳ nhỏ 
    và không thể nhận thấy đối với hầu hết ứng dụng.

7. TW-ANIMATE-CSS
--------------------------------------------------------------------------------
Plugin chuyển đổi các hiệu ứng Animate.css nguyên bản thành các utility classes trong Tailwind CSS.

* ƯU ĐIỂM (PROS):
  - Tải nhanh & dễ dùng: Cung cấp sẵn hàng chục hiệu ứng chuyển động kinh điển (fade, bounce, 
    zoom, slide, v.v.) dưới dạng các class Tailwind đơn giản, không cần cấu hình 
    Keyframes hay Transition phức tạp trong CSS.
  - Tương thích tốt với Tailwind v4: Được import trực tiếp vào CSS nguồn thông qua 
    `@import "tw-animate-css"`, hoạt động mượt mà với bộ biên dịch mới.

* NHƯỢC ĐIỂM (CONS):
  - Thiếu khả năng tùy biến sâu: Vì là các hiệu ứng định sẵn (pre-defined), lập trình viên 
    khó có thể thay đổi chi tiết đường cong bezier hoặc các bước chuyển động phức tạp 
    giữa chừng, không linh hoạt bằng việc tự viết keyframes hoặc dùng GSAP/Motion.
